# 🛠️ Real-time Web Broadcasting & Messaging Steps Note

Laravel 12 + Reverb Websockets စနစ်ဖြင့် **Real-time Notifications** နှင့် **Private Messaging (Chat)** ပြုလုပ်ရမည့် အဆင့်ဆင့် လမ်းညွှန်မှတ်စု ဖြစ်ပါသည်။

---

## 🚀 အဆင့် (၁) - ပင်မ Websocket (Reverb) အား သွင်းယူခြင်း

ပထမဦးစွာ သင့် Terminal တွင် အောက်ပါကွန်မန်းကို ရိုက်ပြီး Laravel ၏ ပင်မ Broadcasting စနစ်ကို သွင်းယူပါ -

```bash
php artisan install:broadcasting
```

> **မှတ်ချက်:** မေးခွန်းများ မေးလာပါက `Laravel Reverb` ကို ရွေးချယ်ပြီး၊ Frontend အတွက် လိုအပ်သော Packages များကို သွင်းရန် `Yes` ဟု ပေးလိုက်ပါ။ ၎င်းသည် `.env` ထဲတွင် `BROADCAST_CONNECTION=reverb` ဟု အလိုအလျောက် ပြောင်းလဲပေးသွားမည် ဖြစ်သည်။

---

## 🔔 အဆင့် (၂) - Real-time Notifications ပြုလုပ်ခြင်း

### ၁။ Database Table ဆောက်ခြင်း
Noti များကို database ထဲတွင် အရင်သိမ်းဆည်းနိုင်ရန် ဇယားဆောက်ပေးရပါမည်။
```bash
php artisan notifications:table
php artisan migrate
```

### ၂။ Notification Class ဆောက်ခြင်း
```bash
php artisan make:notification QuestionLiked
```

`app/Notifications/QuestionLiked.php` ကို ဖွင့်ပြီး အောက်ပါအတိုင်း ပြင်ဆင်ပါ -
```php
public function via(object \$notifiable): array
{
    // DB ထဲမှာသိမ်းပြီး Websocket ပေါ်သို့ပါ တစ်ပြိုင်နက် လွှင့်ထုတ်ရန်
    return ['database', 'broadcast']; 
}

public function toArray(object \$notifiable): array
{
    return [
        'message' => auth()->user()->name . ' liked your question.',
        'user_id' => auth()->id(),
    ];
}
```

### ၃။ Controller မှ Noti လှမ်းပို့ခြင်း (Trigger)
တစ်ယောက်ယောက်က Like သို့မဟုတ် Comment ပေးလိုက်သည့် နေရာတွင် အောက်ပါအတိုင်း ရေးပါ -
```php
receiverUser = question->user; // မေးခွန်းပိုင်ရှင်
\$receiverUser->notify(new QuestionLiked(auth()->user()));
```

### ၄။ Vue Frontend မှ Noti အဝင်လှိုင်းကို ဖမ်းယူနားထောင်ခြင်း
Inertia Vue ဖိုင်၏ `onMounted` ထဲတွင် အောက်ပါအတိုင်း လှမ်းနားထောင်ပါ -
```javascript
import { onMounted } from 'vue';

onMounted(() => {
    // Laravel Noti စနစ်သည် App.Models.User.{id} channel ပေါ်မှ အလိုအလျောက် လွှင့်ပေးပါသည်
    window.Echo.private(`App.Models.User.${currentUserId}`)
        .notification((notification) => {
            console.log('Noti အသစ်ဝင်လာပါပြီ -', notification);
            // Navbar Noti စာရင်းထဲသို့ အပေါ်ဆုံးက တွန်းထည့်ခြင်း
            notificationsArray.value.unshift(notification); 
        });
});
```

## 💬 အဆင့် (၃) - Real-time Private Messaging (Chat) ပြုလုပ်ခြင်း

### ၁။ Message Model နှင့် Table ဆောက်ခြင်း
```bash
php artisan make:model Message -m
```
`database/migrations/xxxx_create_messages_table.php` တွင် ဤအတိုင်း ဖြည့်ပါ -
```php
\$table->foreignId('sender_id')->constrained('users')->onDelete('cascade');
\(table->foreignId('receiver_id')->constrained('users')->onDelete('cascade');\)table->text('message');
\(table->boolean('is_read')->default(false);\)table->timestamps();
```
ပြီးလျှင် `php artisan migrate` ကို ရိုက်ပါ။

### ၂။ Broadcast Event ဆောက်ခြင်း
```bash
php artisan make:event MessageSentEvent
```
`app/Events/MessageSentEvent.php` ထဲတွင် **`implements ShouldBroadcast`** ကို မဖြစ်မနေ ထည့်သွင်းပြီး အောက်ပါအတိုင်း ပြင်ပါ -
```php
class MessageSentEvent implements ShouldBroadcast
{
    use Dispatchable, InteractsWithSockets, SerializesModels;

    public \$message;

    public function __construct(Message \$message) {
        this->message = message->load('sender'); // စာပို့သူ ဒေတာပါ တွဲယူခြင်း
    }

    public function broadcastOn(): array {
        // တစ်ဖက်သား၏ ID သီးသန့်လိုင်းပေါ်သို့ ပစ်လွှင့်ခြင်း
        return [new PrivateChannel('chat.' . \$this->message->receiver_id)];
    }

    public function broadcastAs(): string {
        return 'message.sent';
    }
}
```

### ၃။ Channel လုံခြုံရေး သတ်မှတ်ခြင်း (`routes/channels.php`)
```php
Broadcast::channel('chat.{receiverId}', function (user, receiverId) {
    // လက်ရှိ Login ဝင်ထားသူနှင့် လိုင်း ID ကိုက်ညီမှသာ ဝင်ခွင့်ပြုမည်
    return (int) user->id === (int) receiverId; 
});
```

### ၄။ Controller ဘက်မှ စာသိမ်းပြီး လိုင်းလွှင့်ခြင်း
```php
public function sendMessage(Request \$request, \(receiverId) {\)message = Message::create([
        'sender_id' => auth()->id(),
        'receiver_id' => \$receiverId,
        'message' => \$request->message,
    ]);

    // မိမိဆီ လိုင်းပြန်မလာစေရန် toOthers() တွဲသုံးပါ
    broadcast(new MessageSentEvent(\$message))->toOthers();
    
    return response()->json(\$message->load('sender'));
}
```

### ၅။ Vue Frontend မှ ချက်တင်စာသားများ ဖမ်းယူနားထောင်ခြင်း
`ChatRoom.vue` ဖိုင်ထဲတွင် အောက်ပါအတိုင်း ချိတ်ဆက်ပါ -
```vue
<script setup>
import { onMounted, onUnmounted, ref } from 'vue';

const activeMessages = ref([...props.messages]);

onMounted(() => {
    // 💡 မိမိဆီလာမည့် သီးသန့် Chat လိုင်းအား နားထောင်ခြင်း
    window.Echo.private(`chat.${currentUserId}`)
        .listen('.message.sent', (data) => {
            // လက်ရှိ စကားပြောနေသော တစ်ဖက်လူဆီကလာသည့် စာဖြစ်မှသာ ချက်တင်ဘောက်စ်ထဲ ထည့်ပြပါ
            if (data.message.sender_id === props.receiver.id) {
                activeMessages.value.push(data.message);
            }
        });
});

onUnmounted(() => {
    // စာမျက်နှာပြောင်းသွားပါက လိုင်းအား သန့်ရှင်းစွာ ပြန်ပိတ်ပစ်ခြင်း (အလွန်အရေးကြီး)
    window.Echo.leave(`chat.${currentUserId}`);
});
</script>
```

## 🚀 အဆင့် (၄) - စနစ်အား စတင်မောင်းနှင်ခြင်း (Local Testing)

အထက်ပါ ကုဒ်များအားလုံး ပြီးဆုံးသွားပါက Real-time စနစ် အလုပ်လုပ်ရန် သင့် Terminal (PowerShell) အသစ် နှစ်ခုစီ ထပ်မံဖွင့်ပြီး အောက်ပါ ကွန်မန်းများကို **အမြဲတမ်း ပိတ်မပစ်ဘဲ Run ထားပေးရပါမည်** -

* **Terminal ၁ (Websocket ဆာဗာ မောင်းရန်)**:
  ```bash
  php artisan reverb:start
  ```
* **Terminal ၂ (နောက်ကွယ်က Queue စနစ် ပတ်ရန်)**:
  ```bash
  php artisan queue:listen
  ```
